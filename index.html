WRITE permission
Strongly recommended: grant utxoWallet WRITE permission in MiniHub. With WRITE, sends complete in one tap. In READ mode each build step queues a separate Pending Action you must approve manually — and any approval queue race can leave the History row stuck. Right-click utxoWallet in MiniHub → Write mode.

Node lock state
A Minima node whose wallet vault is password-locked can build transactions but cannot sign them — sends will fail at the signing step. If your node is locked, unlock the vault password (Minima CLI: vault password:<pwd> action:unlock or the dashboard equivalent) before sending.
